Claude Agentic Framework v2.1

Layered Memory System

Four cognitive memory layers — from ephemeral tool output to permanent global knowledge — keeping Claude grounded, context-efficient, and hallucination-free.

Architecture

The 4 Memory Layers

Layer 3
Semantic
global · indefinite · always injected
  • ~/.claude/CLAUDE.md — system instructions
  • ~/.claude/memory/MEMORY.md — global rules
  • ~/.claude/memory/GLOBAL_LEARNINGS.md — patterns
  • ~/.claude/agents/ — 8 global agents
  • ~/.claude/skills/ — 7 global skills
Session start cost ~800 tok
Layer 2
Episodic
per-project · project lifetime · auto-maintained
  • .claude/PROJECT_CONTEXT.md — architecture cache
  • .claude/FACTS.md — verified project facts
  • .claude/MEMORY.md — rolling session history
  • .claude/agents/ — local agent overrides
  • .claude/skills/ — local skill overrides
Session start cost ~4300 tok
Layer 1
Working
session-scoped · ephemeral · survives compaction
  • TaskList — active task state (in-context)
  • compressed_context/ — cold-task summaries
  • Modified file paths — tracked by pre_compact
  • Key decisions — preserved on compaction
Runtime cost ~1000 tok
Layer 0
Sensory
per-turn · ephemeral · no persistence
  • Tool outputs — Bash, Read, Write results
  • File contents — currently read files
  • Web results — fetched content
  • Gone after each response
Cost unlimited / turn

Data Flow

Session Start — Injection Order

HOOKS FIRE IN SEQUENCE → CONTEXT BUILT UP BEFORE FIRST MESSAGE
🌐
CLAUDE.md system-reminder L3 ~800 tok
~/.claude/memory/MEMORY.md global rules always present. Highest-level behavioral instructions. Cannot be overridden by project config.
🏗
session_startup.py → auto_prime.py L2 ~2000 tok
Loads .claude/PROJECT_CONTEXT.md — architecture overview, agents, commands, hooks. Git-hash validated, auto-stale on new commits. Run /prime to regenerate.
✅
inject_facts.py L2 ~1500 tok
Loads .claude/FACTS.md — verified project facts (CONFIRMED, GOTCHAS, PATHS, PATTERNS). Auto-created on first session, auto-populated by hooks. Excludes STALE section.
📋
inject_memory.py L2 ~800 tok
Loads .claude/MEMORY.md — last 5 session summaries. What changed, what was fixed, what was committed. Written by auto_memory_writer at session end.
🗺
repo_map.py L2 ~1000 tok
Only fires if project has ≥200 tracked files. Generates a TreeSitter symbol index ranked by importance — prevents blind file exploration in large codebases.

Session Lifecycle

💬
User Message
UserPromptSubmit hooks
  • kr_mode.py → Korean if enabled
  • auto_prime_inject.py → force /prime if no cache
  • analyze_request.py → Caddy: classify task
  • auto_delegate.py → route if complex
🔧
Tool Call
PreToolUse hooks
  • session_lock_manager.py → prevent concurrency
  • unified-damage-control.py → block dangerous patterns
  • auto_review_team.py → review risky Bash
📤
Tool Result
PostToolUse hooks
  • auto_fact_extractor.py → FACTS.md L2
  • auto_context_manager.py → compress @ 70% L1
  • auto_error_analyzer.py → inject on errors
  • auto_cost_warnings.py → warn at thresholds
  • auto_refine.py → improve writes
🏁
Session End
Stop hooks
  • session_lock_manager.py → release lock
  • auto_memory_writer.py → MEMORY.md L2
  • validate_facts.py → prune FACTS.md L2
  • voice_done.py → audio notification
🗜
Compaction
PreCompact hook
  • pre_compact_preserve.py → preserve L1
  • Reads compressed cold-task summaries, active tasks, modified files, key decisions, git diff → injects as preservation instructions into compaction prompt

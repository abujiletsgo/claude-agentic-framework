데이터 흐름

세션 시작 — 자동 실행 항목

최소화 설계: 프로젝트 아키텍처만 자동 주입. 훅 2개, 최소 오버헤드.

SessionStart 훅 → 첫 메시지 전에 컨텍스트 구성
🌐
~/.claude/CLAUDE.md + MEMORY.md L3 · 항상 ~800 tok
글로벌 행동 규칙 및 크로스 프로젝트 학습. 항상 존재. 프로젝트 설정으로 재정의 불가. 최고 우선순위 지침.
🔒
session_startup.py SessionStart
모든 스킬 7개에 SHA-256 무결성 검사. 변조 시 경고. 세션 잠금 초기화. 가장 먼저 실행되는 게이트.
🏗
auto_prime_inject.py → PROJECT_CONTEXT.md L2 · 자동 ~2000 tok
.claude/PROJECT_CONTEXT.md 로드 — 전체 아키텍처 개요: 에이전트, 명령어, 훅, 구조, 핵심 파일. Git 해시로 검증. 해시 변경 시 오래된 것으로 표시 → /prime으로 갱신.
🗺
repo_map.py L2 · 대형 저장소만 ~1000 tok
프로젝트 파일 ≥200개일 때만 실행. TreeSitter 심볼 인덱스 생성 — 맹목적 파일 탐색 방지. 소형 프로젝트는 건너뜀.
FACTS.md와 MEMORY.md를 자동 주입하지 않는 이유?
이 파일들은 필요시에만 읽힙니다. 프로젝트별 작업 시작 시 Claude가 읽는 것이지, 모든 세션에서 무조건 소비되지 않습니다. 필요 없는 세션에서 ~2300 토큰을 절약합니다.
필요시 접근

메모리 호출 — 언제 & 어떻게

에피소딕 메모리는 관련성이 있을 때 읽힙니다. Claude가 작업 컨텍스트를 기반으로 판단합니다.

FACTS.MD 호출 흐름
💬
이 프로젝트의 코드, 설정, 알려진 패턴 관련 작업
트리거: 프로젝트별 지식이 필요한 작업 (버그 수정, 기능 추가, 명령어 실행, 훅 수정).
📋
Read .claude/FACTS.md 필요시
Claude가 Read 도구로 FACTS.md 읽음. 주입 카테고리: CONFIRMED, GOTCHAS, PATHS, PATTERNS. STALE 섹션은 자동 제외.
✅
신뢰 계층 적용
CONFIRMED 항목이 추론을 재정의. 명령어 실행 전 GOTCHAS 확인. 파일 탐색 전 PATHS 참조. PATTERNS는 검증된 시퀀스에 사용.
MEMORY.MD 호출 흐름
🔄
공백 후 작업 재개 (다른 세션, 기능 계속)
트리거: 이전 작업 계속의 맥락 — 부분적으로 수정된 버그, 진행 중인 기능, 최근 변경 사항 불확실.
📖
Read .claude/MEMORY.md 필요시
MEMORY.md 읽기: 롤링 세션 기록 (최대 30개). 각 항목: 날짜, 커밋 메시지, 변경된 파일, 완료된 작업. Git diff 기반 — 요약이 아닌 실제 데이터.
🎯
컨텍스트 정립: 무엇이 변경됐는지, 무엇이 수정됐는지, 어떤 결정이 내려졌는지
최근 항목으로 프로젝트 상태 파악. 항목은 사실적 (git diff + 커밋). 30 세션 후 가장 오래된 항목 자동 삭제.
자동 저장

메모리 저장 — 완전 자동

수동 작업 없음. 훅이 모든 관련 도구 호출 후와 세션 종료 시 백그라운드에서 FACTS.md와 MEMORY.md에 씁니다.

⚡

  • 성공 + 알려진 패턴 → CONFIRMED
    실패 + 오류 패턴 → GOTCHA
    LLM 호출 없음 — 순수 패턴 매칭
📝
모든 Write 후
PostToolUse: Write
  • auto_fact_extractor.py→FACTS.md
  • 핵심 경로 작성 → PATHS 사실

세션 종료
Stop 훅
  • auto_memory_writer.py→MEMORY.md
  • validate_facts.py→FACTS.md 정리
  • MEMORY: git diff + 커밋 메시지.
    변경 사항 없으면 저장 안 함.
    FACTS: 90일 이상 항목 삭제.

End-to-End

전체 요청 라이프사이클

사용자 메시지부터 도구 실행, 메모리 영속화까지 — 모든 터치포인트.

사용자 메시지 → 도구 호출 → 메모리 영속화
1️⃣
사용자 메시지 전송
UserPromptSubmit 실행: kr_mode (한국어?), auto_prime_inject (캐시 없으면 /prime 강제), analyze_request (Caddy 분류: direct/orchestrate/rlm/fusion), auto_delegate (복잡하면 라우팅).
2️⃣
필요시 에피소딕 메모리 읽기 필요시
프로젝트별 작업? → .claude/FACTS.md 읽기 (검증된 사실, 함정, 알려진 경로). 이전 작업 재개? → .claude/MEMORY.md 읽기 (최근 세션 변경 사항). 두 파일 모두 Read 도구로 읽기 — 필요시만 소비.
3️⃣
에이전트 및 스킬 선택
Claude 해결: 로컬 .claude/agents/ 먼저 확인 → 글로벌 ~/.claude/agents/ 폴백. 스킬도 동일. Caddy 권고 참고. 복잡한 작업 → orchestrator가 팀 소환. 단순 작업 → 직접 실행.
4️⃣
PreToolUse가 모든 도구 호출 게이팅
session_lock_manager: 동시 파일 쓰기 방지. unified-damage-control: 파괴적 패턴 차단. auto_review_team: 실행 전 위험 Bash 검토.
5️⃣
도구 실행
Bash / Read / Write / Edit 실행. 도구 출력이 Layer 0 (감각)에 착지 — 이번 턴에만 가시.
6️⃣
PostToolUse: 메모리 + 관측성 토큰 0
auto_fact_extractor: FACTS.md에 CONFIRMED/GOTCHA/PATHS 사실 기록. auto_context_manager: 70% 컨텍스트에서 냉각 작업 압축. auto_error_analyzer: Bash 실패 시 진단 주입. 모두 디스크에 씀 — 토큰 비용 없음.
7️⃣
세션 종료 → Stop 훅이 메모리 영속화
auto_memory_writer: git diff → 세션 요약을 .claude/MEMORY.md에 기록. validate_facts: 90일 이상 오래된 사실 삭제. voice_done: 오디오 알림. 다음 세션을 위한 메모리 파일 준비.

What you get

Key Features

Everything is automatic. Install once, and the framework handles context management, routing, memory, and safety for every session.

🔀
Auto-Routing
Caddy classifies every prompt and picks the right strategy — no manual `/orchestrate` needed for most tasks.
🧠
Persistent Memory
Project facts auto-extracted after tool calls. Session summaries written on stop. Tagged with git author for team visibility.
🛡️
Damage Control
Pattern-based guardrails block dangerous commands and protect key files — even in Yolo mode.
🔌
Circuit Breakers
Hooks that fail 3× auto-disable for 60s. No runaway loops or cascading errors.
📦
Context Compaction
At 70% context, cold tasks are compressed. At compaction, summaries + active state are preserved automatically.
🔭
RepoMap
For repos ≥200 files, TreeSitter generates a ranked symbol index at session start — instant codebase orientation.
💡
Knowledge Pipeline
Learnings extracted from tool calls, stored in SQLite FTS5. Top-5 relevant entries injected at session start.
🏷️
Team Attribution
Every fact and session memory entry is tagged with the git author — see who verified what and when across your team.
Get started

Quick Install

One script installs everything: validates hooks, generates ~/.claude/settings.json, creates symlinks, and regenerates docs.

# Clone and install
$ git clone <repo> ~/Documents/claude-agentic-framework
$ cd ~/Documents/claude-agentic-framework && bash install.sh
# On any new project, prime the context cache
$ /prime → creates .claude/PROJECT_CONTEXT.md
# Edit config — never touch settings.json directly
$ vim templates/settings.json.template && bash install.sh
Never edit ~/.claude/settings.json directly. Edit templates/settings.json.template and run bash install.sh to apply changes. The script regenerates README.md + CLAUDE.md from repo state on every run.
Architecture

Directory Structure

Everything lives in one repo. Configuration is template-driven; docs are auto-generated.

global-hooks/
25 hooks across 6 event types. Core automation pipeline.
global-agents/
8 agent definitions. Opus/Sonnet/Haiku tier assignments.
global-skills/
7 skills / slash commands for engineering tasks.
global-commands/
13 slash commands symlinked into ~/.claude/commands/.
templates/
settings.json.template — edit this, not settings.json.
data/
model_tiers.yaml, caddy_config.yaml, knowledge-db/.
apps/observability/
Vue 3 + Bun dashboard. Ports 4000 (API) / 5173 (UI).
scripts/
generate_docs.py auto-generates README + CLAUDE.md.
Global vs Local resolution: Local files at .claude/agents/ or .claude/skills/ always override globals with the same name. Use local overrides to customize behaviour per project without touching the framework source.
Before you start

Critical Rules

These four rules prevent the most common mistakes. Violating them can cause permanent data loss or broken sessions.

MUST KNOW
🚫
Never delete a hook file referenced in live settings.json
The current session caches config at startup. If you delete a hook script mid-session, the Stop hook will error infinitely. Instead: replace the file with a no-op stub, then delete after running bash install.sh in a new session.
📝
Edit templates/settings.json.template, not settings.json
~/.claude/settings.json is generated. Your changes will be overwritten on next install. Always edit the template and run bash install.sh.
🐍
All Python runs via uv run
Not python3, not pip, not poetry. Every hook script uses #!/usr/bin/env -S uv run --script — inline dependency declarations, zero virtualenv management.
⚡
Circuit breakers self-heal — don't fight them
If a hook keeps failing, it auto-disables for 60s. State lives in ~/.claude/circuit_breakers/. Delete the state file to reset early. Check logs before hacking around a disabled hook.

제공 기능

주요 기능

모든 것이 자동입니다. 한 번 설치하면 프레임워크가 모든 세션의 컨텍스트 관리, 라우팅, 메모리, 안전성을 처리합니다.

🔀
자동 라우팅
Caddy가 모든 프롬프트를 분류하고 올바른 전략을 선택합니다 — 대부분의 작업에서 수동 /orchestrate가 불필요합니다.
🧠
지속적 메모리
툴 호출 후 프로젝트 팩트 자동 추출. 세션 종료 시 요약 저장. 팀 가시성을 위해 git 저자로 태깅.
🛡️
데미지 컨트롤
패턴 기반 가드레일이 위험한 커맨드를 차단하고 핵심 파일을 보호합니다 — Yolo 모드에서도 마찬가지입니다.
🔌
서킷 브레이커
3번 실패하는 훅은 60초 동안 자동 비활성화됩니다. 무한 루프나 연쇄 오류가 발생하지 않습니다.
📦
컨텍스트 압축
컨텍스트가 70%에 도달하면 오래된 작업이 압축됩니다. 압축 시 요약과 활성 상태가 자동 보존됩니다.
🔭
RepoMap
200개 이상 파일의 저장소에서 TreeSitter가 세션 시작 시 랭킹된 심볼 인덱스를 생성합니다 — 즉각적인 코드베이스 파악.
💡
지식 파이프라인
툴 호출에서 학습 내용을 추출해 SQLite FTS5에 저장합니다. 세션 시작 시 상위 5개 관련 항목이 자동 주입됩니다.
🏷️
팀 기여 추적
모든 팩트와 세션 메모리 항목이 git 저자로 태깅됩니다 — 팀 전체에서 누가 무엇을 언제 검증했는지 확인 가능합니다.
시작하기

빠른 설치

하나의 스크립트로 모든 것을 설치합니다: 훅 유효성 검사, ~/.claude/settings.json 생성, 심링크 생성, 문서 재생성.

# 클론 및 설치
$ git clone <repo> ~/Documents/claude-agentic-framework
$ cd ~/Documents/claude-agentic-framework && bash install.sh
# 새 프로젝트에서 컨텍스트 캐시 초기화
$ /prime → .claude/PROJECT_CONTEXT.md 생성
# 설정 편집 — settings.json을 직접 수정하지 마세요
$ vim templates/settings.json.template && bash install.sh
~/.claude/settings.json을 직접 편집하지 마세요. templates/settings.json.template을 편집하고 bash install.sh를 실행하여 변경사항을 적용하세요. 스크립트는 실행 시마다 저장소 상태에서 README.md와 CLAUDE.md를 재생성합니다.
아키텍처

디렉토리 구조

모든 것이 하나의 저장소에 있습니다. 설정은 템플릿 기반이며 문서는 자동 생성됩니다.

global-hooks/
6가지 이벤트 타입에 걸친 25개 훅. 핵심 자동화 파이프라인.
global-agents/
8개 에이전트 정의. Opus/Sonnet/Haiku 티어 할당.
global-skills/
엔지니어링 작업을 위한 7개 스킬 / 슬래시 커맨드.
global-commands/
~/.claude/commands/에 심링크된 13개 슬래시 커맨드.
templates/
settings.json.template — settings.json이 아닌 이 파일을 편집하세요.
data/
model_tiers.yaml, caddy_config.yaml, knowledge-db/.
apps/observability/
Vue 3 + Bun 대시보드. 포트 4000 (API) / 5173 (UI).
scripts/
generate_docs.py가 README와 CLAUDE.md를 자동 생성합니다.
글로벌 vs 로컬 우선순위: .claude/agents/ 또는 .claude/skills/의 로컬 파일은 동일한 이름의 글로벌 파일을 항상 오버라이드합니다. 프레임워크 소스를 건드리지 않고 로컬 오버라이드를 사용해 프로젝트별 동작을 커스터마이즈하세요.
시작 전

필수 규칙

이 네 가지 규칙은 가장 흔한 실수를 방지합니다. 위반 시 영구적인 데이터 손실이나 세션 오류가 발생할 수 있습니다.

반드시 알아야 할 사항
🚫
라이브 settings.json에 참조된 훅 파일을 절대 삭제하지 마세요
현재 세션은 시작 시 설정을 캐시합니다. 세션 중 훅 스크립트를 삭제하면 Stop 훅이 무한 오류를 발생시킵니다. 대신: 파일을 no-op 스텁으로 교체한 후, 새 세션에서 bash install.sh를 실행한 뒤 삭제하세요.
📝
settings.json이 아닌 templates/settings.json.template을 편집하세요
~/.claude/settings.json은 생성된 파일입니다. 변경사항은 다음 설치 시 덮어씌워집니다. 항상 템플릿을 편집하고 bash install.sh를 실행하세요.
🐍
모든 Python은 uv run으로 실행합니다
python3도, pip도, poetry도 사용하지 않습니다. 모든 훅 스크립트는 #!/usr/bin/env -S uv run --script를 사용합니다 — 인라인 의존성 선언, 가상환경 관리 불필요.
⚡
서킷 브레이커는 자동 복구됩니다 — 억지로 우회하지 마세요
훅이 계속 실패하면 60초 동안 자동 비활성화됩니다. 상태는 ~/.claude/circuit_breakers/에 저장됩니다. 빠른 초기화를 원하면 상태 파일을 삭제하세요. 비활성화된 훅을 억지로 우회하기 전에 로그를 먼저 확인하세요.

Claude Agentic Framework

훅 레퍼런스

6가지 이벤트 타입에 걸쳐 26개의 훅. 모든 훅은 자동으로 실행됩니다 — 별도 호출이 필요 없습니다. 훅은 디스크에 기록하고 컨텍스트를 주입하여 프레임워크의 자동화 근간을 형성합니다.

개요 에이전트 라우팅 스킬 훅 메모리
설계

훅 아키텍처

훅은 Claude Code 이벤트에 응답하여 서브프로세스로 실행되는 Python 스크립트입니다. 컨텍스트를 읽고, 파일에 기록하고, 내용을 주입할 수 있지만, additionalContext를 반환하지 않는 한 Claude API 토큰을 소비하지 않습니다.

제로 토큰 훅 (대다수)

대부분의 훅은 디스크에 기록하고 {}를 반환합니다. 토큰 비용이 없습니다. 예시: auto_fact_extractor (FACTS.md 기록), auto_memory_writer (MEMORY.md 기록), session_lock_manager (락 상태 업데이트).

# 예시: 제로 토큰 훅 출력
print(json.dumps({}))  # ← additionalContext 없음
sys.exit(0)

컨텍스트 주입 훅

additionalContext를 반환하는 훅은 텍스트를 Claude 세션에 주입합니다. 토큰을 소비하지만 신뢰할 수 있는 정보를 제공합니다. 예시: auto_prime_inject (PROJECT_CONTEXT.md), auto_error_analyzer (오류 진단).

# 예시: 컨텍스트 주입 훅 출력
print(json.dumps({
  "additionalContext": "## Context\n..."
}))
sys.exit(0)
서킷 브레이커가 PostToolUse 훅을 보호합니다. 훅이 3번 실패하면 60초 동안 자동 비활성화됩니다. 상태는 ~/.claude/circuit_breakers/에 저장됩니다. 멈춘 훅을 수동으로 재설정하려면 상태 파일을 삭제하세요.
핵심 규칙: 현재 settings.json이 참조하는 훅 파일을 절대 삭제하지 마세요. 파일이 없으면 Stop 훅이 무한 오류를 발생시킵니다. 대신, 파일을 no-op 스텁으로 교체한 후 다음 세션에서 bash install.sh를 실행하여 설정을 깨끗하게 재생성하세요.

SessionStart 훅 2개 — Claude Code 세션이 열릴 때 한 번 실행

—

repo_map.py

프로젝트의 TreeSitter 심볼 인덱스를 생성합니다. 프로젝트에 추적된 파일이 200개 이상일 때만 실행됩니다. ~/.claude/REPO_MAP.md에 순위가 매겨진 심볼 인덱스를 생성합니다. 대규모 코드베이스에서의 맹목적인 파일 탐색을 방지합니다.

auto_prime_inject.py

PostToolUse 훅 11개 — 각 도구 호출이 완료된 후 실행

Bash

기록 후 정제 패스를 트리거합니다. 기록된 파일에서 명백한 문제(구문 오류, 누락된 임포트, 미완성 구현)를 확인하고 다음으로 넘어가기 전에 Claude가 수정하도록 유도합니다.
auto_dependency_audit.py
Write|Edit
파일 변경 후 새로운 의존성이 추가되었는지, 설치되어 있는지 확인합니다. package.json / requirements.txt에 없는 패키지를 임포트하는 코드를 작성하는 일반적인 실수를 잡아냅니다.
auto_context_manager.py

PreCompact 훅 1개 — Claude Code가 컨텍스트를 압축하려 할 때 실행
pre_compact_preserve.py
—
컨텍스트 보존. 압축 전에 다음을 읽습니다: ~/.claude/data/compressed_context/의 압축된 작업 요약, 활성 TaskList 항목, 최근 수정된 파일, 주요 결정 사항, git diff stat. 이 모든 내용을 압축 프롬프트에 보존 지침으로 주입하여 Claude가 요약 시 무엇을 유지해야 할지 알 수 있도록 합니다.
설정

훅 관리

모든 훅은 templates/settings.json.template에 정의됩니다. ~/.claude/settings.json을 직접 편집하지 마세요 — 설치 시 덮어쓰여집니다.

훅 추가하기

# 1. 훅 스크립트 생성
global-hooks/framework/my-hook.py

# 2. 템플릿에 추가
# templates/settings.json.template
{
  "matcher": "Bash",
  "hooks": [{
    "type": "command",
    "command": "uv run __REPO_DIR__/global-hooks/framework/my-hook.py",
    "timeout": 5
  }]
}

# 3. 적용
bash install.sh

안전하게 훅 제거하기

# 잘못된 방법: 파일을 즉시 삭제하면
# Stop 훅이 모든 응답에서 오류를 발생시킵니다.
#
# 올바른 방법: 먼저 스텁으로 교체하세요.
# 파일 내용을 no-op 스크립트로 바꾸고,
# 템플릿에서 제거한 뒤 bash install.sh를 실행하면
# 그 후 파일을 안전하게 삭제할 수 있습니다.
훅이 멈춘 경우: 서킷 브레이커가 비활성화했습니다. ~/.claude/circuit_breakers/[hookname].json을 삭제하여 재설정하세요. 또는 60초를 기다리면 자동으로 복구됩니다.

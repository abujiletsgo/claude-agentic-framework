Claude Agentic Framework

Hook Reference

26 hooks across 6 event types. Every hook fires automatically — no invocation needed. Hooks write to disk and inject context, forming the automation backbone of the framework.

Design

Hook Architecture

Hooks are Python scripts that run as subprocesses in response to Claude Code events. They can read context, write files, and inject content — but they don't consume Claude API tokens unless they return additionalContext.

Zero-token hooks (majority)

Most hooks write to disk and return {}. They have no token cost. Examples: auto_fact_extractor (writes FACTS.md), auto_memory_writer (writes MEMORY.md), session_lock_manager (updates lock state).

# Example: zero-token hook output
print(json.dumps({}))  # ← no additionalContext
sys.exit(0)

Context-injecting hooks

Hooks that return additionalContext inject text into Claude's session. These cost tokens but provide authoritative information. Examples: auto_prime_inject (PROJECT_CONTEXT.md), auto_error_analyzer (error diagnosis).

# Example: context-injecting hook output
print(json.dumps({
  "additionalContext": "## Context\n..."
}))
sys.exit(0)
Circuit breakers protect PostToolUse hooks. If a hook fails 3 times, it auto-disables for 60 seconds. State stored in ~/.claude/circuit_breakers/. Delete the state file to reset a stuck hook manually.
Key rule: Never delete a hook file that live settings.json references. The Stop hook errors infinitely if the file is missing. Instead: replace the file with a no-op stub, then run bash install.sh in the next session to regenerate config cleanly.
Full Reference

All 26 Hooks

SessionStart 2 hooks — fires once when Claude Code session opens
session_startup.py
—
SHA-256 integrity check on all 7 skill files. Initializes session locks. Detects tampered skills and warns. Runs first — acts as security gate for the session.
repo_map.py
—
Generates a TreeSitter symbol index for the project. Only fires if project has ≥200 tracked files. Produces a ranked symbol index at ~/.claude/REPO_MAP.md. Prevents blind file exploration in large codebases.
UserPromptSubmit 4 hooks — fires before Claude processes each message
kr_mode.py
—
Korean language mode. If activated via /kr, translates the incoming prompt to English using Haiku, then Claude responds in Korean. Toggle with /kr skill.
auto_prime_inject.py
—
Injects PROJECT_CONTEXT.md into session as authoritative project knowledge. If no cache exists, forces a /prime run. If git hash changed since last prime, marks context as potentially stale.
analyze_request.py
—
Caddy classifier. Analyzes prompt complexity, quality requirement, scope, and domain. Uses keyword matching + optional Haiku call. Writes classification to temp file for auto_delegate to read.
auto_delegate.py
—
Reads Caddy classification and injects routing recommendation into context. Returns: strategy (direct/team/orchestrate/rlm/fusion), confidence %, relevant skills, guidance note.
PreToolUse 3 hooks — fires before each tool call executes
session_lock_manager.py
Read|Edit|Write
File conflict detection. Acquires a lock before file operations. If another session holds the lock, returns a block with details. Prevents two parallel Claude sessions from corrupting the same file.
unified-damage-control.py
Bash|Edit|Write
The safety layer. Blocks dangerous commands matching a YAML pattern file. Protects: force-push to main, dropping production databases, editing ~/.ssh, writing to system paths. Edit the patterns file to customize.
auto_review_team.py
Bash
Team coordination hook for risky Bash commands. Before high-risk commands execute, triggers a lightweight review. Only fires for commands above a risk threshold — not every Bash call.
PostToolUse 11 hooks — fires after each tool call completes
session_lock_manager.py
*
Releases file locks acquired by PreToolUse. Runs after every tool call to ensure locks don't persist if a tool errors out.
context-bundle-logger.py
Bash|Write|Edit
Save-game system. Snapshots session state (modified files, active tasks, key decisions) after meaningful tool calls. Enables session restore via /loadbundle.
auto_cost_warnings.py
*
Tracks cumulative API cost in the session. Injects warnings at configurable thresholds (e.g., $1, $5, $10). Prevents surprise bills on long autonomous sessions.
auto_error_analyzer.py
Bash
Only fires when a Bash command fails (exit code ≠ 0). Analyzes the error output, identifies root cause pattern, and injects a diagnosis into context. Claude gets the diagnosis alongside the raw error.
auto_refine.py
Write|Edit
Triggers a refinement pass after writes. Checks the written file for obvious issues (syntax, missing imports, incomplete implementations) and prompts Claude to fix them before moving on.
auto_dependency_audit.py
Write|Edit
After file changes, checks if new dependencies were introduced and whether they're installed. Catches the common case of writing code that imports a package not in package.json / requirements.txt.
auto_context_manager.py
Bash|Write|Edit
Context health monitor. Every 10 turns, checks context usage. At 70%+, scans for "cold" tasks (not mentioned in 20+ turns) and writes compressed summaries to ~/.claude/data/compressed_context/. Prevents context death on long sessions.
auto_voice_notifications.py
Bash|Write|Edit
Audio notifications for key events (task complete, error, waiting for input). Uses macOS say command. Works best when running long autonomous tasks — you can step away and get notified.
auto_team_review.py
Write|Edit
Post-write review trigger. After significant file changes, checks if the changes warrant a team review cycle. Part of the builder+validator pattern for team mode tasks.
auto_fact_extractor.py
Bash|Write
Memory write hook. Zero LLM calls — pure pattern matching. Bash success + known pattern → CONFIRMED fact. Bash failure → GOTCHA. Write to key path → PATHS fact. Tags each entry with date + git author. Writes to .claude/FACTS.md.
+ 1 more (circuit_breaker_wrapper)
wraps above
All PostToolUse hooks except session_lock_manager are wrapped by circuit_breaker_wrapper.py. If a hook fails 3 times, it's disabled for 60s. Prevents a broken hook from blocking every tool call.
Stop 5 hooks — fires when Claude finishes responding
session_lock_manager.py
—
Final cleanup — releases all locks held by this session. Runs first in Stop to unblock any waiting sessions.
check_lthread_progress.py
—
Validates RLM (recursive agent) state. If an RLM job was in progress, checks whether it completed or stalled. Reports incomplete l-threads to prevent silent failures.
auto_memory_writer.py
—
Memory write hook. Checks git diff — if files changed, writes a dated session summary to .claude/MEMORY.md. Entry includes: date + @author, commit message, changed files, completed task summaries. Only writes if something actually happened — idle sessions produce no entry.
validate_facts.py
—
FACTS.md maintenance. Prunes entries in the STALE section that are older than 90 days. Active facts (CONFIRMED/GOTCHAS/PATHS/PATTERNS) are never auto-deleted — they persist indefinitely. Only the junk pile gets cleaned.
voice_done.py
—
Audio "done" notification when the session ends. Useful for long autonomous tasks — signals that Claude has finished and is waiting.
PreCompact 1 hook — fires when Claude Code is about to compact the context
pre_compact_preserve.py
—
Context preservation. Before compaction, reads: compressed task summaries from ~/.claude/data/compressed_context/, active TaskList items, recently modified files, key decisions, and git diff stat. Injects all of this as preservation instructions into the compaction prompt — so Claude knows what to keep when summarizing.
Configuration

Managing Hooks

All hooks are defined in templates/settings.json.template. Never edit ~/.claude/settings.json directly — it gets overwritten on install.

Adding a hook

# 1. Create the hook script
global-hooks/framework/my-hook.py

# 2. Add to template
# templates/settings.json.template
{
  "matcher": "Bash",
  "hooks": [{
    "type": "command",
    "command": "uv run __REPO_DIR__/global-hooks/framework/my-hook.py",
    "timeout": 5
  }]
}

# 3. Apply
bash install.sh

Removing a hook safely

# WRONG: deleting the file immediately causes
# the Stop hook to error on every response.
#
# RIGHT: stub it first
# Replace file contents with a no-op script,
# remove from template, run bash install.sh,
# then delete the file safely.
If a hook is stuck: Circuit breaker disabled it. Delete ~/.claude/circuit_breakers/[hookname].json to reset. Or wait 60 seconds — it auto-recovers.

스킬 & 도구

스킬은 재사용 가능하고 슬래시 커맨드로 호출할 수 있는 워크플로우입니다. 전체 엔지니어링 라이프사이클을 위한 7가지 프레임워크 스킬과 함께, 전역 스킬을 덮어쓰는 프로젝트별 스킬을 생성하는 기능도 제공합니다.

스킬이란?

스킬 vs 에이전트

스킬과 에이전트는 서로 다른 문제를 해결합니다. 어느 것을 사용할지 알면 과도한 설계를 방지할 수 있습니다.

스킬 /slash-command

현재 Claude 세션에서 실행되는 프롬프트 템플릿입니다. 별도의 컨텍스트나 에이전트 생성이 없습니다. 특정 지시 집합을 불러와 실행하는 "매크로"로 생각하면 됩니다.

  • ✓ 인라인 실행 — 컨텍스트 오버헤드 없음
  • ✓ 현재 대화 히스토리에 접근 가능
  • ✓ /skill-name으로 호출
  • ✓ 적합한 용도: 알려진 워크플로우를 가진 집중적인 작업

에이전트 Task tool

Task 도구를 통해 생성되는 독립된 컨텍스트를 가진 서브프로세스입니다. 독립적으로 실행되고 결과를 반환합니다. 부모 에이전트는 자식 에이전트의 탐색 내용을 컨텍스트에 축적하지 않습니다.

  • ✓ 격리된 컨텍스트 — 컨텍스트 오염 없음
  • ✓ 다른 에이전트와 병렬 실행 가능
  • ✓ 적합한 용도: 대규모 탐색, 병렬 작업
  • ✓ 파일 내용이 아닌 결과를 반환
경험 법칙: 무엇을 해야 할지 정확히 알 때는 스킬을 사용하세요. 먼저 탐색이 필요할 때는 에이전트를 사용하세요.
프레임워크 스킬

7가지 기본 스킬

bash install.sh에 의해 ~/.claude/skills/에 설치됩니다. 모든 프로젝트에서 사용 가능합니다.

code-review
/code-review [file or PR]
PR 수준의 코드 리뷰. 버그, 보안 문제, 성능 문제, 스타일 위반을 검사합니다. 심각도별 구조화된 결과를 반환합니다: critical → major → minor → suggestion.
예시
/code-review src/auth/session.ts
검사 항목: SQL 인젝션, 안전하지 않은 JWT 설정, 입력 유효성 검사 누락, 경쟁 조건
/code-review — review the last 3 changed files
error-analyzer
/error-analyzer [paste error or describe]
오류, 예외, 크래시에 대한 근본 원인 분석. 스택 트레이스를 읽고 원인을 파악하며 구체적인 수정 방법을 제안합니다. FACTS.md GOTCHAS를 자동으로 교차 참조합니다.
예시
/error-analyzer
스택 트레이스를 붙여넣으면 → 반환됩니다: 근본 원인, 영향받은 파일:행, 수정 제안, 검증 방법
facts new
/facts [subcommand]
프로젝트의 FACTS.md를 관리합니다 — 환각을 방지하는 살아있는 사실 시트. 서브커맨드: list, add CATEGORY "text", stale "pattern", summary, purge-stale.
주요 사용 방법
/facts list
날짜 및 작성자와 함께 카테고리별 모든 사실을 표시
/facts add CONFIRMED "deploy command is: npm run deploy:prod"
오늘 날짜와 git 사용자 이름으로 검증된 사실을 추가
/facts stale "webpack"
webpack 관련 사실을 STALE로 이동 (예: Vite로 마이그레이션 후)
왜 중요한가: 사실은 STALE 더미에서만 삭제됩니다 (90일 후). CONFIRMED/GOTCHAS/PATHS 사실은 무기한 유지됩니다 — 2년 동안 잘 작동하는 프로젝트는 모든 지식을 보존합니다.
refactoring-assistant
/refactoring-assistant [target]
안전하고 점진적인 리팩토링. 대상을 분석하고 변경 가능한 부분을 파악하며, 작고 안전한 단계의 순서를 제안하고 각 단계 사이에 테스트를 수행하며 실행합니다.
예시
/refactoring-assistant UserService class
1단계: 인터페이스 추출 → 2단계: UserService + UserRepository로 분리 → 3단계: 호출자 업데이트 → 각 단계 검증
security-scanner
/security-scanner [path or scope]
OWASP Top 10 및 일반적인 패턴에 대한 취약점 감사. 검사 항목: 인젝션, 인증 취약점, XSS, 안전하지 않은 역직렬화, 잘못된 설정. 심각도와 해결 방법이 포함된 결과를 반환합니다.
예시
/security-scanner src/api/
CRITICAL: 검색 엔드포인트의 SQL 인젝션 (47번째 줄) — 원시 쿼리에서 사용자 입력이 정제되지 않음
MAJOR: config.ts에 JWT 시크릿 하드코딩 — 환경 변수로 이동 필요
test-generator
/test-generator [file or function]
포괄적인 테스트 스위트를 생성합니다. 대상 코드를 읽고 엣지 케이스, 오류 경로, 정상 경로를 파악합니다. 프로젝트의 기존 테스트 프레임워크(Jest, Vitest, pytest 등)로 테스트를 작성합니다.
예시
/test-generator src/utils/price.ts
생성 항목: formatPrice()의 단위 테스트, 엣지 케이스 (null, 음수, 오버플로우), 로케일 테스트, 통화 표시를 위한 스냅샷
knowledge-db
/knowledge-db [subcommand]
세션 간 학습을 위한 SQLite FTS5 지식 데이터베이스. 여러 프로젝트에 걸친 결정, 패턴, 솔루션을 저장하고 검색합니다. FACTS.md와 별개로 — 이것은 크로스 프로젝트 지식을 위한 것입니다.
예시
/knowledge-db search "prisma connection pooling"
반환됩니다: 다른 프로젝트에서 작업한 Prisma 설정에 관한 과거 학습 내용
/knowledge-db store "always set connectionLimit: 5 in Prisma for Lambda"
글로벌 vs 로컬

글로벌 vs 로컬 스킬

글로벌 스킬은 어디서나 작동합니다. 로컬 스킬은 프로젝트별로 존재하며 — 같은 이름의 글로벌 스킬보다 우선합니다.

글로벌 스킬

~/.claude/skills/ (install.sh에 의해 심링크됨)

7가지 프레임워크 스킬. 모든 프로젝트에서 사용 가능합니다. 이름으로 호출됩니다: /code-review, /facts 등.

로컬 스킬

.claude/skills/ (프로젝트 디렉토리)

프로젝트별로 존재합니다. 같은 이름의 글로벌 스킬을 덮어씁니다. 로컬 code-review 스킬은 글로벌 스킬 위에 프로젝트별 규칙을 추가합니다.

로컬 스킬 만들기

# .claude/skills/deploy.md
---
name: deploy
description: >
  Deploy this application to production. Handles environment-specific
  config, database migrations, and health checks.
---

# Deploy

Run the production deployment sequence for this project:

1. Run tests: `npm run test:ci`
2. Build: `npm run build`
3. Run migrations: `npx prisma migrate deploy`
4. Deploy to Fly.io: `fly deploy --config fly.production.toml`
5. Verify health: `curl https://api.ourapp.com/health`

If any step fails, stop and report the error. Do not proceed to the next step.

After successful deploy, add a CONFIRMED fact: "Last deploy: [date]"
/deploy로 호출하면 — Claude가 스킬을 불러와 위의 정확한 순서를 실행합니다. 커맨드를 외울 필요도, 설정 파일을 찾을 필요도 없습니다. 스킬 자체가 런북입니다.

로컬 스킬은 팀 컨벤션을 코드화하는 가장 좋은 방법입니다. 팀의 모든 사람이 기본적으로 동일한 워크플로우를 사용할 수 있습니다.

PR 체크리스트 스킬

# .claude/skills/pr-check.md
Before creating a PR, verify:
- Tests pass: npm test
- No console.log statements in src/
- API changes documented in docs/api.md
- Migration has a down() function
- New env vars added to env.example

환경 설정 스킬

# .claude/skills/setup.md
Set up local development:
1. Copy env.example to env file
2. docker compose up -d postgres redis
3. npm install
4. npx prisma migrate dev
5. npm run seed
6. npm run dev (port 3000)
